# Totem 移动数据库系统介绍

## 什么是 Totem

Totem 是一个基于**对象代理模型**的移动数据库系统，使用 **Java** 语言编写，运行在 **Android 平台**上。

与纯关系数据库不同，对象代理模型的核心思想是：除了普通的**源类**，还可以定义**代理类**——一种从源类通过规则派生出的类。代理类与源类之间有双向指针（BiPointerTable）和属性映射（SwitchingTable）维护依存关系。当源类数据发生 INSERT/UPDATE/DELETE 时，系统通过**更新迁移**机制自动同步到所有依赖的代理类。

## 开发环境：Android Studio

### 什么是 Android Studio

Android Studio 是 Google 官方的 Android 应用 IDE，基于 JetBrains IntelliJ IDEA，提供代码编辑、编译构建、模拟器运行、调试的一站式功能。Totem 数据库是一个 Android 应用，你需要使用 Android Studio 来编译和运行它。

### 安装与配置

1. 从 [developer.android.com/studio](https://developer.android.com/studio) 下载 Android Studio（Windows / macOS / Linux 均可）  
2. 安装时保持默认选项，安装程序会自动下载 Android SDK
3. 首次启动后，在 **SDK Manager**（Tools → SDK Manager）中确认已安装 **Android API 35** 或更高版本
4. 在 **AVD Manager**（Device Manager → Create Device）中创建一个虚拟设备（推荐 Pixel 系列，系统镜像选 API 35）

### 获取与打开项目

1. 克隆项目代码：
   ```
   git clone https://github.com/MarchSeventh258/totem_mobile
   ```
2. 启动 Android Studio，选择 **Open**，选中 `totem_mobile` 目录
3. 等待 Gradle 同步完成（首次可能需要几分钟下载依赖）
4. 在顶部工具栏选择刚才创建的虚拟设备，点击绿色 **Run** 按钮（或 Shift+F10）
5. 等待编译安装完成，模拟器上会出现 Totem 数据库的界面

### 项目结构

```
totem_mobile/
  app/src/main/java/edu/whu/tmdb/          ← 数据库内核源码
       ├── query/                            ← 查询执行层
       │   └── operations/impl/              ← SQL 语句实现
       ├── storage/                           ← 存储引擎（LSM-Tree）
       │   ├── memory/SystemTable/           ← 系统表定义
       │   ├── level/                         ← SSTable / Compaction
       │   └── cache/                         ← 多级缓存
       ├── Log/                               ← Write-Ahead Log
       └── MainActivity.java                 ← Android 前端界面
  app/libs/                                   ← 第三方 JAR（JSqlParser 等）
```

- **`edu/whu/tmdb/`**：数据库内核代码，所有实验修改均在此目录下
- **`MainActivity.java`**：Android 前端界面，包含 SQL 输入框和结果展示区域，可自行修改美化
- **JSqlParser**：SQL 语法解析器，以 JAR 包形式打包引入，负责将 SQL 字符串解析为 Java 语句对象。如需修改 SQL 语法（新增关键字或子句），需修改 JSqlParser 源码：修改 `JSqlParserCC.jjt` 语法文件后 `./gradlew build` 重新编译，再将生成的 JAR 替换 `app/libs/` 下的旧版本

## 系统已有功能

### SQL 解析与执行

- 支持 `CREATE CLASS`、`DROP CLASS`、`INSERT`、`DELETE`、`UPDATE`、`SELECT`
- SELECT 查询支持 WHERE 过滤、JOIN（INNER / LEFT / RIGHT / FULL / NATURAL）、GROUP BY、ORDER BY、LIMIT
- 支持 UNION、INTERSECT、EXCEPT 集合操作
- 聚合函数支持 AVG、MIN、MAX、COUNT、SUM（需配合 GROUP BY 使用）

### 存储引擎

- 基于 LSM-Tree 的分层存储引擎（7 层），SSTable 文件包含 Zone Map、Bloom Filter 和 B-Tree 索引
- 基于 LRU 的多级缓存（DataCache / MetaCache）
- Write-Ahead Log（WAL）崩溃恢复

## 对象代理模型

### 四种代理类

| 代理类型 | 语法 | 功能 |
|---------|------|------|
| SelectDeputy | `CREATE SELECTDEPUTY ... AS SELECT ... WHERE ...` | 从源类选择部分行和列，类似视图 |
| JoinDeputy | `CREATE JOINDEPUTY ... AS SELECT ... FROM A, B WHERE ...` | 多表连接派生 |
| UnionDeputy | `CREATE UNIONDEPUTY ... AS SELECT ... UNION SELECT ...` | 合并多个源类实例 |
| GroupDeputy | `CREATE GROUPDEPUTY ... AS SELECT ... GROUP BY ...` | 分组聚合派生 |

### 七张系统表

对象代理模型的依存关系通过以下系统表维护：

#### 1. ClassTable —— 类注册表

| 字段 | 类型 | 说明 |
|------|------|------|
| `classid` | int | 类唯一 ID |
| `classname` | String | 类名 |
| `attrnum` | int | 属性总数 |
| `classtype` | int | 类型：0=源类或SelectDeputy, 1=JoinDeputy, 2=UnionDeputy, 3=GroupDeputy |

#### 2. AttributeTable —— 属性定义表

| 字段 | 类型 | 说明 |
|------|------|------|
| `classid` | int | 所属类 ID |
| `attrid` | int | 属性 ID |
| `attrname` | String | 属性名 |
| `attrtype` | String | 属性类型 |
| `alias` | String | 别名 |
| `isVirtual` | int | 是否为虚拟属性 |

#### 3. ObjectTable —— 对象-元组映射表

| 字段 | 类型 | 说明 |
|------|------|------|
| `classid` | int | 所属类 ID |
| `tupleid` | int | 元组 ID |
| `sstSuffix` | int | 元组所在 SSTable 文件后缀 |

#### 4. DeputyTable —— 代理关系表

| 字段 | 类型 | 说明 |
|------|------|------|
| `originid` | int | 源类 ID |
| `deputyid` | int | 代理类 ID |
| `ruleid` | int | 代理规则 ID（关联 DeputyRuleTable） |

#### 5. DeputyRuleTable —— 代理规则表

| 字段 | 类型 | 说明 |
|------|------|------|
| `ruleid` | int | 规则 ID |
| `deputyrule` | String[] | 规则数组 |

#### 6. BiPointerTable —— 双向指针表

| 字段 | 类型 | 说明 |
|------|------|------|
| `classid` | int | 源类 ID |
| `objectid` | int | 源类元组 ID |
| `deputyid` | int | 代理类 ID |
| `deputyobjectid` | int | 代理类元组 ID |

这条记录表示「源类 classid 的 objectid 号元组」对应「代理类 deputyid 的 deputyobjectid 号元组」。通过查这张表可以：
- 从源元组找到所有对应的代理元组
- 从代理元组反向追溯到源元组

#### 7. SwitchingTable —— 属性映射表

| 字段 | 类型 | 说明 |
|------|------|------|
| `oriId` | int | 源类 ID |
| `oriAttrid` | int | 源属性 ID |
| `oriAttr` | String | 源属性名 |
| `deputyId` | int | 代理类 ID |
| `deputyAttrId` | int | 代理属性 ID |
| `deputyAttr` | String | 代理属性名 |
| `rule` | String | 映射规则 |

## 对象代理模型使用示例

以下是一个完整的示例，展示从创建源类到定义四种代理类的完整流程。

### 准备：创建源类并插入数据

```sql
-- 创建歌手类
CREATE CLASS Singer (
    id INT, name STRING, sex STRING, age INT,
    nationality STRING, company STRING
);

-- 创建歌曲类
CREATE CLASS Song (
    id INT, name STRING, singer STRING, date INT
);

-- 创建照片类
CREATE CLASS Photo (
    id INT, name STRING, singer STRING, date INT
);

-- 插入数据
INSERT INTO Singer VALUES
(0, 'TaylorSwift', 'F', 36, 'America', 'ATVMusic'),
(1, 'EdSheeran', 'M', 33, 'UK', 'WarnerMusic'),
(2, 'JayChou', 'M', 45, 'China', 'JVR');

INSERT INTO Song VALUES
(0, 'Red', 'TaylorSwift', 2012),
(1, '1989', 'TaylorSwift', 2014),
(2, 'Divide', 'EdSheeran', 2017);

INSERT INTO Photo VALUES
(0, 'Fearless', 'TaylorSwift', 2008),
(1, 'HeadAboveWater', 'AvrilLavigne', 2022);
```

### SelectDeputy：选择部分行和列

```sql
-- 只保留 2014-2022 年的歌曲，且只投影 name, singer, date 三列
CREATE SELECTDEPUTY song_2014_2022 AS
SELECT name, singer, date FROM Song
WHERE date >= 2014 AND date <= 2022;

SELECT * FROM song_2014_2022;
-- 结果：1989(TaylorSwift, 2014), Divide(EdSheeran, 2017)
```

### UnionDeputy：合并多个源类

```sql
-- 将 Song 和 Photo 的 name, singer, date 合并到一起
CREATE UNIONDEPUTY song_photo AS
SELECT name, singer, date FROM Song
UNION
SELECT name, singer, date FROM Photo;

SELECT * FROM song_photo;
-- 结果包含 Song 的 3 条 + Photo 的 2 条
```

### JoinDeputy：多表连接

```sql
-- 将 Song 和 Singer 按歌手名连接
CREATE JOINDEPUTY singer_song AS
SELECT Song.name, Song.date, Song.singer,
       Singer.sex, Singer.age, Singer.nationality, Singer.company
FROM Song, Singer
WHERE Song.singer = Singer.name;

SELECT * FROM singer_song;
-- 结果每条记录同时包含歌曲信息和歌手信息
```

### GroupDeputy：分组聚合

```sql
-- 按年份统计歌曲数量
CREATE GROUPDEPUTY year_songnumber AS
SELECT date, COUNT(id) as song_count
FROM Song
GROUP BY date;

SELECT * FROM year_songnumber;
-- 结果：2012 1首, 2014 1首, 2017 1首
```

### 跨类查询（路径表达式）

跨类查询使用 `->` 箭头语法在代理关系链上导航，是对象代理模型独有的查询方式：

```sql
-- 从 Singer 通过 singer_song 代理类导航，取出 Song 的 name
SELECT Singer -> singer_song.name FROM Singer;

-- 在起始类上添加筛选条件
SELECT Singer{name = 'TaylorSwift'} -> singer_song.name FROM Singer;

-- 在中间代理类上添加筛选条件
SELECT Singer -> singer_song{date >= 2014} -> Song.name FROM Singer;

-- 复合条件 + 逆向导航
SELECT Singer{sex = 'F' AND age > 35} -> singer_song -> Song.name FROM Singer;
```

语法说明：
- `ClassName{条件}`：筛选该类的元组
- `-> deputyName`：沿 BiPointerTable 指针导航到代理类
- `.attrName`：投影目标类的指定属性

## 更新迁移机制

对象代理模型的一个重要特性是：当源类数据变化时，代理类会自动同步。

```sql
-- 创建代理类
CREATE SELECTDEPUTY RecentSongs AS
SELECT name, singer, date FROM Song WHERE date >= 2014;

SELECT * FROM RecentSongs;  -- 1989, Divide

-- 插入新歌 → 自动同步到代理类
INSERT INTO Song VALUES (3, 'Midnights', 'TaylorSwift', 2022);
SELECT * FROM RecentSongs;  -- 新增 Midnights

-- 更新歌曲 → 代理类同步更新
UPDATE Song SET name = '1989(TaylorsVersion)' WHERE id = 1;
SELECT * FROM RecentSongs;  -- 名称已更新

-- 删除歌曲 → 代理类同步删除
DELETE FROM Song WHERE id = 0;
SELECT * FROM RecentSongs;  -- Red 已消失
```

对于 GroupDeputy，插入/删除/更新源数据后聚合值会自动重算，代理元组会相应更新。

## Android 前端界面

系统提供简易的 Android 前端界面（`MainActivity.java`），包含：
- **SQL 输入框**：输入 SQL 语句
- **执行按钮**：点击执行

运行方式：在 Android Studio 中点击 Run，等待编译安装到模拟器，即可在模拟器中看到界面。你可以直接使用现有界面，也可以修改 `MainActivity.java` 进行美化或功能扩展。
