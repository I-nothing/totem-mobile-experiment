## 任务一：实现非严格 SelectDeputy 代理类

### 背景

目前系统已支持 **严格 SelectDeputy**，即创建时必须通过 WHERE 子句指定代理规则，系统根据规则自动判定每条源数据是否属于代理类：

```sql
CREATE CLASS Student (id INT, name STRING, score INT);
INSERT INTO Student VALUES (1, 'Alice', 90), (2, 'Bob', 50);

-- 严格模式：规则写在 WHERE 里，系统自动判定
CREATE SELECTDEPUTY GoodStudent AS
SELECT name, score FROM Student WHERE score > 80;

SELECT * FROM GoodStudent;
-- Alice(90) 自动进入，Bob(50) 被过滤
```

本次任务要求你实现 **非严格 SelectDeputy**：创建时不指定 WHERE 条件，元组是否属于代理类不由系统自动判定，而是在 INSERT 时由用户**显式声明**该元组应放入哪个代理类：

```sql
CREATE CLASS Student (id INT, name STRING, score INT);

-- 非严格模式：无 WHERE 条件
CREATE SELECTDEPUTY GoodStudent AS
SELECT name, score FROM Student;

-- 插入时手动指定哪些元组属于 GoodStudent
INSERT INTO Student VALUES (1, 'Alice', 90) INTO GoodStudent;
INSERT INTO Student VALUES (2, 'Bob', 50);   -- 不放入 GoodStudent

SELECT * FROM Student;      -- Alice(90), Bob(50)
SELECT * FROM GoodStudent;   -- 只有 Alice(90)
```

核心区别：**严格代理类由规则驱动，非严格代理类由用户手动标注。**「谁属于代理类」的决策权从系统交给了人。

### 要求

#### 1.1 支持非严格 SelectDeputy 的创建

- 创建 SelectDeputy 时，如果 SELECT 语句不包含 WHERE 子句，则视为非严格模式
- 非严格代理类创建时不填充数据，初始为空
- 系统需要能够区分严格模式与非严格模式

##### 示例

```sql
CREATE CLASS Student (id INT, name STRING, score INT);

-- 非严格代理类（无 WHERE）
CREATE SELECTDEPUTY GoodStudent AS
SELECT name, score FROM Student;

-- 未指定 INTO 时仅插入源类，不传播到非严格代理类
INSERT INTO Student VALUES (1, 'Alice', 90);
SELECT * FROM GoodStudent;
-- 期望：
-- name   | score

-- 显式指定 INTO，元组同时插入源类和代理类
INSERT INTO Student VALUES (2, 'Bob', 85) INTO GoodStudent;
SELECT * FROM GoodStudent;
-- 期望：
-- name   | score
-- Bob    | 85
```

#### 1.2 实现非严格代理的显式插入

- 当 INSERT 语句指定了 `INTO deputyClass` 时，元组除插入源类外还应显式插入到指定代理类中，并建立双向指针
- 若指定的代理类并非该源类的非严格代理类，应报错
- 当 INSERT 未指定 INTO 时，元组仅插入源类，不自动传播到非严格代理类
- 严格代理类的现有行为保持不变

##### 示例

```sql
CREATE CLASS Student (id INT, name STRING, score INT);

-- 严格代理类（有 WHERE）
CREATE SELECTDEPUTY GoodStudent AS
SELECT name, score FROM Student WHERE score > 80;

-- 对严格代理类使用 INTO 应报错
INSERT INTO Student VALUES (1, 'Alice', 90) INTO GoodStudent;
-- 期望：报错，提示不允许对严格代理类使用 INTO
```

```sql
CREATE CLASS Student (id INT, name STRING, score INT);
CREATE CLASS Teacher (id INT, name STRING, subject STRING);

CREATE SELECTDEPUTY GoodStudent AS
SELECT name, score FROM Student;

INSERT INTO Teacher VALUES (1, 'Smith', 'Math') INTO GoodStudent;
-- 期望：报错
```

#### 1.3 处理更新迁移

- DELETE 迁移：删除源元组时，若该元组关联了非严格代理类，应同步删除代理类中的对应元组
- UPDATE 迁移：更新源元组时，若关联了非严格代理类，应同步更新代理类中的对应属性值

##### 示例

```sql
CREATE CLASS Student (id INT, name STRING, score INT);

CREATE SELECTDEPUTY GoodStudent AS
SELECT name, score FROM Student;

INSERT INTO Student VALUES (1, 'Alice', 90) INTO GoodStudent;
INSERT INTO Student VALUES (2, 'Bob', 50);
INSERT INTO Student VALUES (3, 'Charlie', 85) INTO GoodStudent;

-- 删除关联了代理类的源元组
DELETE FROM Student WHERE id = 1;
SELECT * FROM GoodStudent;
-- 期望：Alice 消失，只剩 Charlie

-- 删除未关联代理类的源元组
DELETE FROM Student WHERE id = 2;
SELECT * FROM GoodStudent;
-- 期望：不受影响，仍为 Charlie
```

```sql
CREATE CLASS Student (id INT, name STRING, score INT);

CREATE SELECTDEPUTY GoodStudent AS
SELECT name, score FROM Student;

INSERT INTO Student VALUES (1, 'Alice', 90) INTO GoodStudent;
INSERT INTO Student VALUES (2, 'Bob', 50);

-- 更新关联了代理类的元组
UPDATE Student SET score = 95 WHERE id = 1;
SELECT * FROM GoodStudent;
-- 期望：score 同步更新（Alice | 95）

-- 更新未关联代理类的元组，代理类不受影响
UPDATE Student SET score = 60 WHERE id = 2;
SELECT * FROM GoodStudent;
-- 期望：不受影响（Alice | 95）
```

## 任务二：为 GroupDeputy 代理类实现 HAVING 子句支持

### 背景

GROUP BY 代理类（GroupDeputy）可以从源类中按分组聚合生成代理元组，例如：

```sql
CREATE CLASS Student (id INT, name STRING, dept STRING, score INT);
INSERT INTO Student VALUES (1, 'Alice', 'CS', 90), (2, 'Bob', 'CS', 70), (3, 'Charlie', 'Math', 85);

CREATE GROUPDEPUTY HighScoreDept AS
SELECT dept, AVG(score) as avg_score, COUNT(*) as cnt
FROM Student
GROUP BY dept
HAVING AVG(score) >= 80;
-- CS 组 avg=80, Math 组 avg=85，均满足条件
```

目前系统的现状：
- **SELECT 查询中的 GROUP BY 可用**，但 HAVING 子句被完全忽略（`Having.java` 是空类，`SelectImpl.java` 中预留了接口但未实现）
- **GroupDeputy 创建时**：SQL 规则被整体存为字符串，但 HAVING 不参与过滤，不符合条件的组也会被创建为代理元组
- **更新迁移时**：INSERT/UPDATE/DELETE 触发的代理类自动更新完全不检查 HAVING 条件。即使聚合值不再满足 HAVING 条件，代理元组也不会被移除或拒绝

### 要求

#### 2.1 实现 SELECT 查询中的 HAVING 过滤

- 目前系统在 GROUP BY 聚合后没有对结果做 HAVING 过滤，需要补充这一环节
- HAVING 过滤应发生在 GROUP BY 聚合之后、ORDER BY 排序之前

##### 示例

```sql
CREATE CLASS Employee (id INT, dept STRING, salary INT);

INSERT INTO Employee VALUES
(1, 'HR', 5000), (2, 'HR', 6000),
(3, 'IT', 8000), (4, 'IT', 9000),
(5, 'Sales', 3000);

SELECT dept, AVG(salary) FROM Employee
GROUP BY dept
HAVING AVG(salary) > 5000;
-- 期望：Sales(avg=3000) 被过滤，HR 和 IT 保留

SELECT dept, AVG(salary) FROM Employee
GROUP BY dept
HAVING AVG(salary) >= 3000;
-- 期望：三个部门都在
```

#### 2.2 实现 GroupDeputy 创建时的 HAVING 过滤

- 创建 GroupDeputy 时，在遍历 SELECT 结果建立代理元组和 BiPointer 条目之前，应先根据 HAVING 条件过滤不满足的组
- 被 HAVING 过滤掉的组不应被插入代理类，也不应创建对应的 BiPointer 条目

##### 示例

```sql
CREATE CLASS Employee (id INT, dept STRING, salary INT);

INSERT INTO Employee VALUES
(1, 'HR', 5000), (2, 'HR', 6000),
(3, 'IT', 8000), (4, 'IT', 9000),
(5, 'Sales', 3000);

CREATE GROUPDEPUTY HighSalaryDept AS
SELECT dept, AVG(salary) as avg_sal, COUNT(*) as cnt
FROM Employee
GROUP BY dept
HAVING AVG(salary) > 5000;

SELECT * FROM HighSalaryDept;
-- 期望：Sales(avg=3000) 不应出现，只有 HR 和 IT
```

#### 2.3 实现更新迁移中的 HAVING 检查

当源类数据变化时，更新迁移机制需要结合 HAVING 条件来决定代理元组的增删。三种情况——满足→不满足（删除）、不满足→满足（创建）、始终满足（更新）。

##### 示例

```sql
CREATE CLASS Employee (id INT, dept STRING, salary INT);

INSERT INTO Employee VALUES
(1, 'HR', 5000), (2, 'HR', 6000),
(3, 'IT', 8000), (4, 'IT', 9000),
(5, 'Sales', 3000);

CREATE GROUPDEPUTY HighSalaryDept AS
SELECT dept, AVG(salary) as avg_sal, COUNT(*) as cnt
FROM Employee
GROUP BY dept
HAVING AVG(salary) > 5000;

-- 已有分组新增条目，聚合值更新但 HAVING 判定不变
INSERT INTO Employee VALUES (6, 'IT', 10000);
SELECT * FROM HighSalaryDept;
-- 期望：IT avg_sal 重算为 9000，cnt 变为 3；HR 不变

-- 原本不满足 HAVING 的分组因新增而满足，应新建代理元组
INSERT INTO Employee VALUES (7, 'Sales', 9000);
SELECT * FROM HighSalaryDept;
-- 期望：Sales avg=(3000+9000)/2=6000 > 5000，应出现
```

```sql
CREATE CLASS Employee (id INT, dept STRING, salary INT);

INSERT INTO Employee VALUES
(1, 'HR', 5000), (2, 'HR', 6000),
(3, 'IT', 8000), (4, 'IT', 9000),
(5, 'Sales', 4000), (6, 'Sales', 8000);

CREATE GROUPDEPUTY HighSalaryDept AS
SELECT dept, AVG(salary) as avg_sal, COUNT(*) as cnt
FROM Employee
GROUP BY dept
HAVING AVG(salary) > 5000;

-- 删除后分组仍满足 HAVING，聚合值更新
DELETE FROM Employee WHERE id = 4;
SELECT * FROM HighSalaryDept;
-- 期望：IT 仅剩 id=3(8000)，仍 > 5000，avg_sal 和 cnt 更新

-- 删除后分组不再满足 HAVING，应删除代理元组
DELETE FROM Employee WHERE id = 6;
SELECT * FROM HighSalaryDept;
-- 期望：Sales 仅剩 id=5(4000)，avg=4000 <= 5000，Sales 代理元组应删除
```

```sql
CREATE CLASS Employee (id INT, dept STRING, salary INT);

INSERT INTO Employee VALUES
(1, 'HR', 5000), (2, 'HR', 6000),
(3, 'IT', 8000), (4, 'IT', 9000);

CREATE GROUPDEPUTY HighSalaryDept AS
SELECT dept, AVG(salary) as avg_sal, COUNT(*) as cnt
FROM Employee
GROUP BY dept
HAVING AVG(salary) > 5000;

-- 更新后分组仍满足 HAVING，聚合值更新
UPDATE Employee SET salary = 9500 WHERE id = 4;
SELECT * FROM HighSalaryDept;
-- 期望：IT avg_sal 更新为 (8000+9500)/2=8750

-- 更新后分组不再满足 HAVING，应删除代理元组
UPDATE Employee SET salary = 3000 WHERE id = 2;
SELECT * FROM HighSalaryDept;
-- 期望：HR avg=(5000+3000)/2=4000 <= 5000，HR 应消失

-- 分组变为满足 HAVING，应新建代理元组
UPDATE Employee SET salary = 9000 WHERE id = 1;
UPDATE Employee SET salary = 9000 WHERE id = 2;
SELECT * FROM HighSalaryDept;
-- 期望：HR avg=(9000+9000)/2=9000 > 5000，HR 重新出现
```
